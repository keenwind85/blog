# 팀의 하루를 요약해주는 AI 비서를 만들다

## 매일 아침, "어제 뭐했더라?"

우리 팀은 5인 이하의 소규모 팀이다. 작은 팀이니까 소통이 쉬울 거라고 생각할 수 있는데, 현실은 좀 다르다.

코드는 GitHub에 있고, 소통은 Slack에서 하고, 디자인은 Figma에서 진행하고, 기획 문서는 Google Drive에 올린다. 4개의 플랫폼에 팀의 활동이 흩어져 있다. 매일 아침 출근하면 이걸 전부 돌아다니면서 파악해야 한다.

- GitHub: 어제 어떤 커밋이 올라왔지? PR 리뷰는 누가 했나?
- Slack: 5개 채널에서 어떤 이야기가 오갔지? 내가 놓친 멘션은?
- Figma: 디자이너가 어떤 화면을 업데이트했나?
- Google Drive: 기획 문서가 새로 올라왔나?

PO로서 팀 전체 상황을 조망하는 게 내 일인데, 이걸 매일 수작업으로 하고 있었다. 솔직히 말하면, 안 하는 날도 있었다. 바쁘면 Slack만 훑고 넘어가고, GitHub 커밋은 PR 올라왔을 때나 확인하고, Figma는 디자이너가 말해주면 그제야 보는 식이었다.

그러다 생각했다.

> "이거 AI가 대신 해주면 안 되나?"

## 구상: AI가 팀 활동을 모아서 브리핑해주자

구상은 단순했다. 4개 플랫폼의 API를 호출해서 어제 팀이 뭘 했는지 데이터를 모으고, AI가 분석해서 요약하고, 그 결과를 Slack에 보내준다. 매일 아침 9시에 자동으로.

```
[매일 오전 9시]
GitHub 활동 수집  ─┐
Slack 메시지 수집  ─┤
Figma 변경 수집   ─┼─→ Claude가 통합 분석 ─→ Slack 브리핑 전송
Drive 작업일지    ─┘
```

여기에 한 가지를 더 얹었다. Google Drive의 팀 공유 폴더는 사람들이 문서를 아무렇게나 올리다 보면 금방 난장판이 된다. 이걸 Gemini한테 시켜서 자동으로 정리하게 하면 어떨까?

그래서 에이전트를 두 개로 나눴다.

| 에이전트 | 실행 시간 | AI 엔진 | 역할 |
|---------|----------|---------|------|
| Drive 서브에이전트 | 매일 오후 5시 | Gemini | Drive 변경 감지 + 폴더 자동 정리 + 작업일지 저장 |
| 메인 에이전트 | 매일 오전 9시 | Claude | 4개 소스 통합 수집 + 분석 + Slack 브리핑 |

오후 5시에 Gemini가 먼저 Drive를 정리하고 작업일지를 남기면, 다음 날 오전 9시에 Claude가 그 작업일지까지 포함해서 전체를 분석하는 구조다. 일종의 **릴레이**다.

## 기술 스택

```
언어: Python 3
AI (브리핑): Claude Max — Claude Code CLI 비대화형 모드
AI (Drive 관리): Gemini 2.5 Flash — 무료 API
데이터 소스: GitHub API, Slack API, Figma API, Google Drive API
알림: Slack Bot
스케줄링: macOS LaunchAgent (오전 9시) + GitHub Actions cron (오후 5시)
```

왜 Python인가? 솔직히 큰 이유는 없다. 4개 API를 호출하고 결과를 문자열로 조합하는 게 전부라서 어떤 언어든 상관없었다. 다만 Python은 requests 라이브러리 하나면 API 호출이 끝나고, AI 관련 예제도 Python이 압도적으로 많아서 바이브 코딩하기 편했다.

Claude를 메인 분석 엔진으로 쓴 이유는 **Claude Max 구독** 때문이다. 이미 Claude Code를 쓰고 있었고, CLI의 비대화형 모드(`claude -p`)를 호출하면 API 비용 없이 분석을 돌릴 수 있다. Gemini를 Drive 서브에이전트로 쓴 건 무료 API 한도가 넉넉하고, Google Drive API와 같은 생태계라서 연동이 자연스러웠기 때문이다.

## 프로젝트 구조

```
ssm-team-daily-agent/
├── config.py                     # 설정 및 KST 시간 유틸
├── main.py                       # 메인 오케스트레이터 (오전 9시)
├── daily_briefing.sh             # LaunchAgent용 실행 스크립트
├── collectors/
│   ├── github_collector.py       # GitHub 커밋, PR, 이슈, 리뷰 수집
│   ├── slack_collector.py        # Slack 메시지, 스레드 수집
│   ├── figma_collector.py        # Figma 코멘트, 버전 히스토리 수집
│   ├── gdrive_collector.py       # Drive 작업일지 읽기
│   └── news_collector.py         # 업계 뉴스 수집
├── analyzers/
│   └── claude_analyzer.py        # Claude CLI로 통합 분석
├── notifiers/
│   └── slack_notifier.py         # Slack 전송
└── cloud_agent/
    └── main.py                   # Gemini Drive 서브에이전트
```

구조는 의도적으로 단순하게 잡았다. `collectors/`가 데이터를 모으고, `analyzers/`가 분석하고, `notifiers/`가 보낸다. 각 collector는 독립적이라서 하나가 실패해도 나머지는 돌아간다.

## 4개 Collector — 각 플랫폼에서 뭘 가져오나

### GitHub Collector

조직 내 전체 저장소에서 어제의 커밋, PR, 이슈를 수집한다. PR은 리뷰 상태까지 가져온다 — 누가 승인했는지, 변경 요청이 있는지, 리뷰 요청을 받고 아직 안 한 사람은 누구인지.

각 항목에는 GitHub 링크가 포함된다. 브리핑을 읽다가 "이 PR 뭐지?" 하면 바로 클릭해서 갈 수 있어야 하니까.

### Slack Collector

지정한 채널들에서 어제 메시지를 수집한다. 여기서 중요한 건 **스레드 답글까지 가져오는 것**이다. Slack에서 진짜 논의는 스레드 안에서 벌어진다. 메인 메시지만 수집하면 "누가 뭐라고 했다" 정도만 알 수 있지, "그래서 결론이 뭔데?"를 놓친다.

멘션도 사용자명으로 치환한다. `<@U12345>`가 아니라 실제 이름이 나와야 브리핑이 읽을 만해진다.

### Figma Collector

다중 프로젝트를 지원하게 만들었다. 파일별 코멘트와 버전 히스토리를 수집한다. 디자이너가 화면을 업데이트하면 버전 히스토리에 남고, 거기에 코멘트를 달면 코멘트도 수집된다.

### Google Drive Collector

이건 직접 Drive를 스캔하는 게 아니다. 오후 5시에 Gemini 서브에이전트가 작성해둔 **작업일지를 읽어오는 것**이다. 작업일지에는 팀원들의 문서 생성/수정/삭제 내역과 서브에이전트가 자동 정리한 결과가 담겨 있다.

### 보너스: News Collector

Claude CLI의 웹 검색 기능으로 돌봄/케어테크 업계 뉴스와 기술 트렌드를 수집한다. 팀에 관련 뉴스를 매일 알려주면, 시장 감각을 유지하는 데 도움이 된다. 출처 URL과 함께 "우리 팀에 적용할 수 있는 점"까지 제안하게 했다.

## Gemini 서브에이전트 — Drive를 자동 정리하는 AI

이 녀석은 GitHub Actions에서 매일 오후 5시에 돌아간다. 하는 일은 세 가지다.

1. **변경 감지**: 팀 공유 드라이브에서 어제 생성/수정/삭제된 문서를 확인하고, 누가 뭘 했는지 기록
2. **폴더 자동 정리**: Gemini가 스스로 규칙을 세워서 문서를 정리
   - 문서 제목을 `[카테고리] 제목_YYYY_MM_DD` 형식으로 통일
   - 폴더 구조를 체계적으로 관리 (생성/이동/병합)
   - 단, 문서 삭제는 못하게 막았다 (실수 방지)
3. **작업일지 저장**: 수행한 내용을 "서브 에이전트 작업일지" 폴더에 날짜별 문서로 저장

처음엔 이 서브에이전트를 Google Cloud Functions로 돌리려 했다. 그런데 Cloud Functions를 쓰려면 GCP 결제 계정을 연결해야 한다. 무료 Tier가 있지만 결제 수단 등록 자체가 필요하다. 이미 GitHub에 코드가 있으니까 **GitHub Actions cron으로 돌리면 무료**다. 바로 전환했다.

## Claude 분석 — 4개 소스를 하나로

4개 collector가 모은 데이터를 Claude에게 넘긴다. 여기서 핵심은 **Claude Code CLI의 비대화형 모드**다.

```bash
claude -p "아래 데이터를 분석해서 팀 일일 브리핑을 작성해줘. ..."
```

`-p` 플래그를 쓰면 프롬프트를 인자로 넘기고 결과를 stdout으로 받을 수 있다. Python의 `subprocess`로 호출하면 스크립트 안에서 Claude를 함수처럼 쓸 수 있는 셈이다.

프롬프트에는 섹션 구분자(`===SECTION===`)를 넣어서 구조화된 출력을 만들게 했다. 팀원별 활동, 미응답/확인 필요 사항, 주요 소통 내용, 제안사항 — 이 네 가지를 분리해서 뽑아낸다. 원문 링크는 분석 결과에도 유지하도록 지시했다. 요약만 보고 끝나는 게 아니라, 원본을 바로 찾아갈 수 있어야 하니까.

## Slack 전송 — 메인은 짧게, 상세는 스레드로

분석 결과를 Slack에 보내는 구조도 고민했다.

```
[메인 메시지] 🤖 팀 일일 브리핑
  └─ 스레드: 👥 팀원별 활동
  └─ 스레드: ⚠️ 미응답/확인 필요
  └─ 스레드: 💬 주요 소통
  └─ 스레드: 💡 제안사항

[메인 메시지] 📰 업계 동향 & 실무 인사이트
  └─ 스레드: 🗒️ 각 출처 및 개별 요약
```

**메인 메시지는 한눈에 파악할 수 있는 요약만** 담는다. 채널에서 스크롤을 많이 차지하면 오히려 안 읽게 된다. 상세 내용은 스레드로 분리해서, 궁금한 사람만 열어서 보게 했다.

업계 뉴스도 별도 메인 메시지로 분리했다. 팀 업무 브리핑과 뉴스는 성격이 다르니까.

## 삽질기

순탄하게 된 건 하나도 없었다.

| 문제                    | 원인                                        | 해결                            |
| --------------------- | ----------------------------------------- | ----------------------------- |
| `claude -p` 실행이 안 됨   | Python subprocess에서 alias가 아닌 별도 환경으로 실행됨 | CLI 전체 경로를 직접 지정              |
| GitHub 계정명-실명 매핑 오류   | AI가 계정명을 보고 실명을 임의로 추측                    | 매핑 로직 제거, 원본 이름 그대로 사용        |
| Slack 메시지가 수집 안 됨     | 봇이 채널에 초대되지 않았음                           | `/invite @봇이름`으로 각 채널에 초대     |
| 시간 경계가 어긋남            | API 기본값이 UTC 기준                           | KST 기준 어제 00:00~23:59로 변환     |
| GCP 결제 필요             | Cloud Functions에 결제 계정 필수                 | GitHub Actions cron으로 대체 (무료) |
| OAuth 클라이언트 CLI 생성 불가 | Google Cloud CLI에서 미지원                    | 브라우저 콘솔에서 수동 생성               |

가장 어이없었던 건 **AI가 사람 이름을 추측한 것**이다. GitHub 계정명이 `wind`면 Claude가 "아마 김○○님일 것 같다"면서 아무 근거 없이 실명을 붙였다. 처음엔 "오 똑똑하네" 했는데, 당연히 틀렸다. 각 플랫폼의 원본 이름을 그대로 쓰는 게 맞다. AI에게 추측을 시키면 안 되는 영역이 있다.

시간대 문제도 클래식한 삽질이었다. 한국에서 일하는데 API가 UTC 기준으로 데이터를 뱉으니까, 오전 9시에 "어제 활동"을 요청하면 UTC 기준 "어제"가 나온다. KST 기준으로 변환해야 진짜 어제다. 9시간 차이가 나는데, 이걸 놓치면 데이터가 반나절씩 밀린다.

## 비용: 거의 0원

이 에이전트를 운영하는 데 추가 비용이 거의 없다.

| 항목             | 사용량           | 무료 한도 대비       |
| -------------- | ------------- | -------------- |
| Claude Max     | 하루 ~10,500 토큰 | 주간 한도의 1~2%    |
| Gemini 무료 API  | 하루 1~2회 호출    | 500 RPD의 1% 미만 |
| GitHub Actions | 월 ~150분       | 2,000분/월의 7.5% |

Claude는 이미 구독 중인 Max 플랜의 CLI를 쓰는 거라 추가 비용이 없다. Gemini도 무료 API로 충분하다. GitHub Actions도 무료 한도 안에서 넉넉히 돌아간다.

스타트업에서 "비용 0원"은 꽤 중요한 포인트다. 아무리 좋은 도구라도 월 몇만 원씩 나가면 "그냥 내가 직접 확인하지 뭐"가 되기 쉽다.

## 스케줄링: macOS LaunchAgent + GitHub Actions

두 에이전트의 실행 환경이 다르다.

**Drive 서브에이전트 (Gemini)** — GitHub Actions cron으로 매일 오후 5시에 실행된다. 클라우드에서 돌아가니까 내 PC가 꺼져 있어도 상관없다.

**메인 에이전트 (Claude)** — 내 Mac의 LaunchAgent로 매일 오전 9시에 실행된다. Claude Max CLI를 호출해야 하니까 로컬에서 돌아야 한다.

```bash
# macOS LaunchAgent 등록
launchctl load ~/Library/LaunchAgents/com.team.dailybriefing.plist

# 수동 테스트
launchctl start com.team.dailybriefing
```

LaunchAgent의 좋은 점은 **9시에 출근해서 PC가 꺼져 있으면, 켜지는 순간 바로 실행**된다는 거다. 출근해서 노트북 열면 잠시 후 Slack에 브리핑이 와 있다.

이상적으로는 메인 에이전트도 클라우드로 올리고 싶지만, Claude Max CLI가 로컬에서만 돌아가는 제약이 있다. API를 직접 쓰면 클라우드로 갈 수 있지만, 그러면 API 비용이 발생한다. 현재로서는 로컬 실행이 가성비 최선이다.

## 만들고 나서 달라진 것

매일 아침 Slack에 브리핑이 와 있다. 열어보면 어제 팀에서 무슨 일이 있었는지 한눈에 보인다.

- 개발자가 어떤 PR을 올렸고, 리뷰가 필요한지
- Slack에서 어떤 논의가 있었고, 아직 답변이 안 된 건 뭔지
- 디자이너가 Figma에서 어떤 화면을 업데이트했는지
- Drive에 새 문서가 올라왔는지

이걸 직접 돌아다니며 확인하던 시간이 사라졌다. 그리고 의외의 효과도 있었다 — **팀원들도 서로의 활동을 더 잘 알게 됐다.** 개발자는 디자이너가 뭘 하는지 몰랐고, 디자이너는 어떤 PR이 올라왔는지 몰랐는데, 브리핑 하나로 전체 그림이 공유된다.

5인 이하의 작은 팀인데도 이런 효과가 있다는 게 좀 놀라웠다. 작은 팀이라고 소통에 문제가 없는 게 아니라, 작은 팀이라서 "굳이 말 안 해도 알겠지" 하며 공유를 건너뛰는 게 문제였던 거다.

## 기획자가 AI 에이전트를 만든다는 것

이 프로젝트는 하루 만에 완성했다. [[백엔드 도전기 1편 - 기획자가 왜 백엔드를 뜯어고치게 되었나|백엔드 리팩토링]]을 하고, [[맞고 1편 - 기획자가 10일 만에 멀티플레이어 카드게임을 만들다|맞고 게임]]을 만들면서 바이브 코딩에 충분히 익숙해진 덕이 크다.

하지만 더 중요한 건, **이게 "개발"이라기보다 "자동화"에 가깝다**는 점이다. 복잡한 알고리즘을 짠 게 아니다. 4개 API를 호출하고, AI에게 분석을 시키고, 결과를 Slack으로 보내는 파이프라인일 뿐이다. 기획자 입장에서 "이런 흐름으로 데이터가 흘러야 한다"는 설계를 할 수 있으면, 나머지는 AI가 코드로 만들어준다.

2026년 지금, AI 에이전트라는 말이 넘쳐나고 있다. 대부분은 거대한 시스템을 이야기한다. 하지만 실제로 현장에서 필요한 건 이런 거다 — 매일 반복하는 단순 작업을 대신해주는 작고 실용적인 에이전트. 거창할 필요 없다. 팀의 하루를 요약해주는 것만으로도 충분히 가치가 있다.

> 🤖 이 글에서 다룬 에이전트의 전체 코드는 사내 프라이빗 저장소에 있습니다. 유사한 에이전트를 만들고 싶은 분은 GitHub API, Slack API, Figma API의 공식 문서를 참고하시면 됩니다. 핵심은 각 API에서 "어제 활동"을 가져오는 것과, 그 결과를 LLM에게 분석시키는 두 단계입니다.
